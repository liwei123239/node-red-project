# ai-assistant — 需求规格

## 概述

集成 OpenAI API，提供实时操作指导、疑难解答、测量数据分析三类 AI 辅助功能，以对话式面板嵌入平台界面。

## 项目信息

- 项目名: node-red-project
- 架构类型: Full-stack TypeScript Monorepo (pnpm + Turborepo)

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-13 | v1   | 初始需求 |

## 用户故事

- 作为学生，我想要在操作过程中获得 AI 实时步骤指引，以便独立完成测量任务
- 作为学生，我想要向 AI 提问设备操作问题，以便在没有老师时也能解决疑难
- 作为科研人员，我想要将测量数据发给 AI 分析，以便快速获得数据处理建议

## 功能需求

1. [F-001] AI 对话面板（侧边栏或浮窗形式），支持多轮对话
2. [F-002] 三种对话模式可切换：操作指导 / 疑难解答 / 数据分析
3. [F-003] 使用 OpenAI Chat Completions API（流式响应）
4. [F-004] 每种模式预置系统提示词，引导 AI 专注于测绘场景
5. [F-005] 数据分析模式允许用户注入当前测量数据作为上下文
6. [F-006] 对话历史在当前 session 内保留，刷新后重置
7. [F-007] AI 响应超时（30s）或错误时显示友好提示并允许重试

## 非功能需求

- 安全: `OPENAI_API_KEY` 仅在服务端使用，不暴露给前端
- 性能: 流式输出，首字节延迟 < 2s
- 可用性: API 不可用时降级显示提示，不影响其他功能

## 验收标准

- [ ] [AC-001] 用户发送消息后，AI 响应以流式方式逐字出现
- [ ] [AC-002] 切换对话模式后，系统提示词更新，对话历史清空
- [ ] [AC-003] 数据分析模式下可注入测量数据，AI 能基于数据作答
- [ ] [AC-004] API Key 未配置时，AI 面板显示配置提示，不崩溃

## 依赖

- `openai` npm 包
- `packages/env` — 新增 `OPENAI_API_KEY` 环境变量
- 依赖 Feature 1 (auth-roles) 完成后集成，确保只有登录用户才能使用 AI
