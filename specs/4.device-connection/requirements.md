# device-connection — 需求规格

## 概述

对接已有硬件网关，通过 HTTP/WebSocket 实现全站仪串口/蓝牙连接管理，提供连接配置 UI 和实时连接状态监控。

## 项目信息

- 项目名: node-red-project
- 架构类型: Full-stack TypeScript Monorepo (pnpm + Turborepo)

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-13 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要通过界面配置连接方式（串口/蓝牙/网络）和参数，以便连接到全站仪
- 作为用户，我想要实时看到设备连接状态，以便知道当前是否可以发送指令
- 作为管理员，我想要管理多套连接配置，以便在不同实训场景快速切换

## 功能需求

1. [F-001] 连接方式支持：串口（RS-232/USB）、蓝牙 SPP、网络（TCP/IP）
2. [F-002] 连接参数配置：IP 地址、端口号、波特率（网络模式）；蓝牙设备名/MAC（蓝牙模式）
3. [F-003] 连接配置持久化到数据库，支持多套配置命名保存
4. [F-004] 点击「连接」后调用硬件网关 API，实时反馈连接结果
5. [F-005] 连接成功后，WebSocket 订阅网关状态推送，页面顶部显示连接状态指示灯
6. [F-006] 支持断开连接操作
7. [F-007] 连接历史日志记录（时间、连接方式、成功/失败原因）

## 非功能需求

- 安全: 硬件网关地址仅在服务端存储，前端不直接访问网关
- 性能: 连接状态更新延迟 < 500ms
- 可用性: 网关不可达时友好提示，不阻塞页面渲染

## 验收标准

- [ ] [AC-001] 配置串口连接参数并点击连接，状态指示灯变绿
- [ ] [AC-002] 断开连接后状态指示灯变红，再次连接恢复
- [ ] [AC-003] 连接配置保存后，刷新页面仍存在
- [ ] [AC-004] 网关不可达时，3 秒内显示错误提示
